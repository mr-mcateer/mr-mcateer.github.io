3. Mobile-First Design Patterns

Every design decision starts at 375px and scales up. The outdoor games category skews heavily mobile, especially during peak buying season when customers browse while at events, parks, and tailgates.

Sticky Nav

Hamburger menu triggers at 768px breakpoint. Transparent on load, solid on scroll with backdrop-filter blur.

Bottom CTA

Bottom-anchored add-to-cart button on product page. Stays in thumb zone for one-handed mobile use.

Swipeable Gallery

Product image gallery supports horizontal swipe on touch devices. Thumbnail nav on desktop, dots on mobile.

Collapsible Sidebar

Research page sidebar nav collapses into an accordion on mobile. Expands above main content on tap.

Touch Targets

All interactive elements meet WCAG 2.1 minimum of 44x44px. Accordion toggles, nav links, and form inputs all sized for fingers.

4. Interaction Patterns

A small set of carefully considered interaction patterns keeps the experience consistent across all pages while maintaining performance.

Pattern Behavior Implementation
Scroll reveal Elements fade up on viewport entry IntersectionObserver with 0.08 threshold, 24px translateY, 0.7s ease-out
Counter animation Stats count up from zero on scroll Ease-out cubic interpolation, triggered by IntersectionObserver on the stats bar
Cart drawer Slide from right with overlay backdrop Fixed panel at right: -420px, slides to right: 0 with .open class; semi-transparent overlay
Accordion One-open-at-a-time within group Native <details>/<summary> with JS toggle that closes siblings on open
Product gallery Thumbnail click swaps main image Click handler on .product-gallery__thumb sets main img src; active border on selected
Email capture Inline validation on blur Regex email pattern check on blur, success/error states via CSS class toggle
Buttons Lift on hover, snap back on press translateY(-1px) on :hover, reset to translateY(0) on :active via CSS transition
Navigation Transparent to solid on scroll Sticky nav with backdrop-filter blur, .site-nav--scrolled class on scroll > 10px

5. Accessibility Checklist

Frisbeam targets WCAG 2.1 Level AA compliance across all pages. The dark theme requires extra attention to contrast ratios and focus indicator visibility.

Requirement Status Notes
Skip link Implemented Hidden anchor revealed on focus, jumps to #main content area
Keyboard navigation Implemented All interactive elements reachable via Tab; accordions operable with Enter/Space
ARIA labels Implemented Nav, cart button, mobile menu, hamburger, breadcrumbs all labeled
Reduced motion Implemented prefers-reduced-motion media query disables all transitions and scroll animations
Color contrast WCAG AA Light text on dark backgrounds meets AA. --text (#F1F5F9) on --bg-primary (#0B0F19) = 15.4:1
Focus indicators Implemented 2px cyan (#00D4FF) outline on :focus-visible with 2px offset for high visibility on dark
Screen reader support Implemented Semantic HTML with proper heading hierarchy, landmark roles, sr-only class for hidden text
Alt text on images Implemented Descriptive alt on content images; decorative images use aria-hidden="true"

6. Performance Budget

Speed is a conversion variable. Every 100ms of additional load time costs roughly 1% in conversion rate. The Frisbeam site is designed to meet aggressive Core Web Vitals targets.

Asset Budget Strategy
HTML <50KB per page Semantic markup, no framework bloat, flat component structure
CSS <30KB single file Design token architecture for minimal redundancy, no utility framework
JavaScript <5KB single file Vanilla JS only, IntersectionObserver for lazy features, no libraries
Images Lazy loaded, CDN-optimized Unsplash CDN with width params (e.g., ?w=800), native loading="lazy" on below-fold images
Fonts Google Fonts CDN display=swap prevents FOIT, preconnect hints reduce connection overhead
LCP target <1.5s on 4G, <3s on 3G Hero image optimized and served from CDN, critical CSS path kept minimal
Performance philosophy: Zero frameworks, zero build tools, zero dependencies. The entire site ships as three static files (HTML, CSS, JS) plus CDN-hosted images and fonts. This keeps the attack surface small, the load times fast, and the maintenance burden close to zero.
